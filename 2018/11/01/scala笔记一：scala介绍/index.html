scala笔记一：scala介绍

Posted on 2018-11-01 | In scala
使用spark进行离线计算，而且spark是scala编写的，因此学习scala

scala文件类型介绍

scala来源于java，因此大部分的逻辑代码是和java相通的，大部分是相似的，但是也有一部分创新的东西。

scala有3类文件格式，实现不同的功能：Object，class 和 Trait（特征）

类是对象的抽象，对象是类的具体实例。类是抽象的，不占内存，对象是具体的，占据内存。

在Object中写main才是实现方法的入口，才能执行方法。一般而言，通过在Object中new class来创建对象调用方法。

class是用于写方法来给object调用的，在scala中可以有多个class和Object和trait。

Object相当于一个java静态类

class相当于一个java普通类

Trait相当于一个java抽象类

scala变量介绍

scala有2个类似于泛型的数据类型：val 和var

如果在声明变量的时候不进行设置变量类型，scala就会根据变量的具体数值进行判断

val在声明赋值之后不许再进行赋值，相当于java中final声明的变量

var是可以进行重新赋值的，相当于java中普通的变量

scala数据类型

scala可以在声明变量的时候进行数据类型的固定：val a:Int = 1

scala修饰符

scala有三类修饰符，public protect private

用 private 关键字修饰，带有此标记的成员仅在包含了成员定义的类或对象内部可见。

在 scala 中，对保护（Protected）成员的访问比 java更严格一些。因为它只允许保护成员在定义了该成员的的类的子类中被访问。而在java中，用protected关键字修饰的成员，除了定义了该成员的类的子类可以访问，同一个包里的其他类也可以进行访问。

Scala中，如果没有指定任何的修饰符，则默认为 public。这样的成员在任何地方都可以被访问。

集合

List

Scala 列表类似于数组，它们所有元素的类型都相同，但是它们也有所不同：列表是不可变的，值一旦被定义了就不能改变，其次列表 具有递归的结构（也就是链接表结构）而数组不是。。

Set

Scala 集合分为可变的和不可变的集合。
默认情况下，Scala 使用的是不可变集合，如果你想使用可变集合，需要引用 scala.collection.mutable.Set 包。

Map

Map 有两种类型，可变与不可变，区别在于可变对象可以修改它，而不可变对象不可以。
默认情况下 Scala 使用不可变 Map。如果你需要使用可变集合，你需要显式的引入 import scala.collection.mutable.Map 类

Tuple

与列表一样，元组也是不可变的，但与列表不同的是元组可以包含不同类型的元素。
Tuple2 是指2种类型数据，可变
