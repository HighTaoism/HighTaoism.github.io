代码问题四：ORA-01722:invalid number

Posted on 2018-06-06 | In oracle
今天系统发现了一个bug，用户通过系统界面调用方法的时候，发现这个方法有的数据能够查出来，有的数据传参的时候会报异常。然后开始查找bug原因，最后总算发现了原因。

问题描述

系统调用一个方法，有的参数传过去能够正常返回数据，有的参数传过去会报错。由于系统的分布式的，服务器比较多，因此通过dubbo的provider的配置文件和tomcat的bin下面的catAlina.sh下面的配置文件对比，最后发现可以调用这个方法的服务器有8台。

没办法，只能把这些机器的日志全部打开，然后传值会报异常的数据给这个方法，最后终于发现问题报错信息：Cause:java.SQLSyntaxErrorException:ORA-01722:invalid number

问题查找原因

  • 1.查看报错的sql，发现没有问题。

  • 2.执行报错的sql，按照前台传的参数放入sql中，没有问题

    那么问题到底是什么？没办法，这个是同事写的代码，不太清楚逻辑，只能跟代码了

    发现：系统查询数据是先查找redis，若是redis服务器中没有数据的话，查找数据库，然后把之前redis没有的数据放进去。

    查看代码好像没有发现问题，逻辑正常的。但是系统invalid number报错，肯定是代码的问题。部分数据传参有问题，那么查redis部分不可能有错误，否则就不会部分数据有问题了。

    于是把查询数据库的部分代码copy出来，编写一个模拟传参的数据，把代码处理的最后参数打印出来

    最后发现：参数的问题。本来是应该传一个org_id的数据，是number类型的，结果传的却是一个参数字符型’CUST’,oracle数据库是不能够隐式转换的，把值放在sql测试，果然是这个错误，问题终于找到。
